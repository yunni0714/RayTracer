# 기물 분류 체계 (사용자 정본 멘탈 모델)

> 출처: 사용자가 업로드한 `Ray.md`. **메이커(사용자)의 정본 분류** — 어드민 면별(per-face) behavior 모델의 어휘 기준이다.
> 주의: 이 분류는 개념 모델이며 현재 코드의 29개 `PieceType`과 **1:1로 매핑되지 않는다**. 상세 면별 특성은 비워져 있고, **어드민 면별 에디터에서 사용자가 직접 채운다**(`docs/ADMIN_PANEL.md`).

## 큰 3분류

기물은 **1. 레이저 · 2. 거울 · 3. 목표** 세 축으로 구분되며, 단계가 올라갈수록 축이 **섞인다**.

### 1. 레이저
- **일반** — 한 방향으로 빔 발사.
- **집광기** — 레이저를 받으면 사출부에서 레이저 발사(일부 목표 특성 지님).
- **가변 추출기** — 반대로, 레이저를 받으면 사출부 발사 **중단**(일부 목표 특성 지님).

### 2. 거울
> 모든 거울은 반사면 입사각 = 반사각, 예외 없음.
- **90도**: 일반 / 양면 / 단면 → (단면 표적)
- **평면**: 일반 / 양면 / 단면
- **45도**: 일반 / 양면 / 단면

### 3. 목표
- **터널** — 터널이 놓인 타일은 **무조건 관통해야** 클리어 조건 만족.
  - **일반** — 특정 면에서 들어오는 레이저를 차단.
  - **방향 터널** — 한쪽 **방향**(면 아님)에서 들어오는 레이저를 제외하고 차단.
  - **긴 터널** — 일반 터널의 대우: 딱 한 형태의 레이저만 통과.
  - **교차 터널** — `+` 방향으로 통과. 단, **두 방향 모두** 통과해야 조건 만족.
  - **관문** — 앞의 수신부를 맞춰야 옆의 터널로 레이저 통과 가능.
- **표적** — 수신부에 레이저가 맞아야 클리어.
  - **일반 표적**
  - **소형** — 수신부 외 면은 레이저 투과.
  - **전방위 표적** — 어떤 각도에서든 맞으면 만족.

### 기타
- **블럭** — 기물 배치 불가, 레이저는 통과.
- **높은 블럭** — 기물 배치 불가, 레이저 차단.

## 단계별 혼합 경향
- **초급**: 세 분류가 섞이지 않음 (예외: 표적거울).
- **중급**: 세 분류가 가장 활발히 섞임 — 기본 2개, 많게는 3개 특성 혼합.
- **고급**: 새 기물 소개 없음. 중·초급 일부에 90° 고정회전 대신 **45° 회전** 추가.

## 면별 모델로의 함의 (요약)
- "터널 = 관통해야 클리어" → 면 효과 `pass + satisfy`.
- "표적거울" → 거울면 `reflect`, 표적면 `reflect + satisfy` (복합).
- "집광기/추출기" → `emit` + 조건부(`conditional`) + 표적면 일부.
- "단면/양면/45°" → 면별 `reflect`/`split` + `surfaceAngle` + `rotationStep`.
- 상세 스키마: `docs/ADMIN_PANEL.md` "면별 효과 모델".
